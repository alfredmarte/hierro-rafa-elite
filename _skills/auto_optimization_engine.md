# [SKILL: AUTO-OPTIMIZATION ENGINE (PHASE OMEGA)]

**Propósito:** Este es tu núcleo biológico. Está diseñado para que purges el despilfarro cognitivo, aprendas de los errores de programación que cometiste, y reescribas el documento `GEMINI.md` para ser más inteligente en la próxima sesión.

## 1. El Diagnóstico Post-Mortem
Antes de concluir un proyecto, debes ejecutar estas preguntas en tu memoria temporal:
- **Token Check:** *¿Usé demasiado chat conversacional inútil? ¿Copié archivos enteros cuando debí usar Diffs?*
- **Fricción Técnica:** *¿Hubo alguna falla de React o Tailwind que tuve que corregir dos veces?*
- **El Acierto (Win):** *¿Qué solución gráfica o de Copy le gustó especialmente al Agency Owner hoy?*

## 2. La Ley de Refactorización Muta-Génica
Si durante el *Diagnóstico Post-Mortem* descubres una oportunidad de Mejora (Win) o un Fallo Grave Estructural (Error), ESTÁS AUTORIZADO a alterar tu propio código maestro (`GEMINI.md`).

- **Reducción de Fluff:** Si notas que una regla en el `GEMINI.md` es redundante, bórrala o abréviala usando tus herramientas de manipulación de archivos (`replace_file_content`).
- **Instaurar Lecciones:** Si solucionaste un error de layout complejo (ej. un problema en Fixed Navbars), inyecta esa micro-regla en el bloque `<CONVERSION_ARCHITECTURE>` de tu `GEMINI.md`.

## 3. Formato de Parche Obligatorio
Si modificas el `GEMINI.md`, debes imprimir en consola el siguiente registro militar para informarle al Agency Owner por qué evolucionaste la mente de la agencia:
```bash
[PHASE OMEGA LOG] 
- Error/Hallucination Detected: [x]
- Solution Learned: [y]
- Tokens Saved / Rule Rewritten: [z] 
"GEMINI.md ha sido actualizado. El sistema es ahora 1 iteración más inteligente."
```
